City Temperature Web App Documentation

1. Introduction

This project is a simple and interactive web application that allows users to search for any city and instantly view its current temperature. The app fetches real-time weather data using the OpenWeatherMap API and displays it in a user-friendly interface.

2. Objectives

  • To create a responsive and engaging web app for checking city temperatures.
  • To practice integrating third-party APIs in web development.
  • To demonstrate the use of HTML, CSS, and JavaScript for building interactive web applications.

3. Methodology

  1. Design the user interface using HTML and CSS for a clean and engaging look.
  2. Use JavaScript to handle user input, fetch data from the OpenWeatherMap API, and update the UI dynamically.
  3. Implement error handling for invalid city names or network issues.
  4. Test the application in different browsers for compatibility.

4. Hardware and Software Required

Hardware:
  • A computer with internet access.
Software:
  • A modern web browser (e.g., Chrome, Firefox, Edge).
  • A text editor or IDE (e.g., Visual Studio Code).
  • (Optional) Local web server for advanced testing.

5. Expected Outcome

  • Users can enter a city name and receive the current temperature in Celsius.
  • The app displays error messages for invalid input or network errors.
  • The interface is visually appealing and responsive.

6. References

  • OpenWeatherMap API Documentation
  • MDN Web Docs (HTML, CSS, JavaScript)

7. Conclusion

This project demonstrates how to build a functional and visually engaging web application using core web technologies and external APIs. It provides a practical example of real-time data fetching and dynamic UI updates, serving as a foundation for more complex web projects.
