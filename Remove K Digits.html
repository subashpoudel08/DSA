Remove K Digits | Greedy + Stack

Given a numeric string and integer k, remove exactly k digits to form the smallest possible number.

Greedy Stack Monotonic O(N) Strings
Reading time: ~9 min Keywords: remove k digits, smallest number, monotonic stack, greedy

Introduction

Problem: Given a string num of digits 0-9 and an integer k, remove exactly k digits so that the resulting number is as small as possible. If the result has leading zeros, strip them; if all digits are removed, return "0".

Summary

  • Maintain a monotonic increasing stack of digits from left to right.
  • While the current digit is smaller than the stack top and k > 0, pop from the stack (remove larger, earlier digits).
  • If any removals remain after traversal, pop from the end (largest suffix digits).
  • Build the string from the stack, strip leading zeros; return "0" if empty.
  • Time complexity is effectively O(N); each digit is pushed and popped at most once.

Example & Breakdown

Example: num = "1432219", k = 3.

  • A non-optimal removal like removing 4, 3, and one 2 can yield "1229".
  • The optimal strategy removes 4, 3, and the first 2, yielding "1219".
  • Why optimal? Removing larger digits that occur earlier shrinks the number most at higher place values.
Greedy insight: Prefer smaller digits in the most significant positions by deleting preceding larger peaks.

Brute-Force Baseline

  • Enumerate all ways to remove k positions from num and take the minimum of the resulting integers.
  • There are C(n, k) possibilities; generating and comparing each is exponential in practice.
  • Even with pruning, this is infeasible for typical constraints; motivates a linear-time greedy approach.

Greedy Criterion & Invariant

The outputâ€™s digits should be as small as possible, as early as possible. Maintain the invariant that the container of chosen digits is non-decreasing:

  • Whenever a new digit d is less than the last chosen digit and k > 0, remove the last chosen digit.
  • This locally optimal removal is globally optimal because earlier positions have higher significance.
  • If no more beneficial removals are possible during the scan, any remaining k must be spent by removing a suffix (largest on the right).

Single-Pass Stack Algorithm

  • Traverse digits left to right with a stack (or dynamic string).
  • While k > 0 and the stack isnâ€™t empty and stack.top() > currentDigit, pop.
  • Push the current digit.
  • After traversal, if k > 0, pop the last k digits.
  • Build the answer from the stack and strip leading zeros; if empty, return "0".

Using a string as a stack (push_back/pop_back) is common and efficient.

Pseudocode & Implementations

function removeKdigits(s, k):
    stack = empty list
    for each digit d in s:
        while stack not empty and k > 0 and stack.back() > d:
            stack.pop_back()
            k -= 1
        stack.push_back(d)

    # remove remaining k from the end (largest suffix)
    while k > 0 and stack not empty:
        stack.pop_back()
        k -= 1

    # build and strip leading zeros
    res = string from stack
    remove leading zeros from res

    if res is empty:
        return "0"
    return res

C++ Implementation

#include <bits/stdc++.h>
using namespace std;

string removeKdigits(string num, int k) {
    string st;
    st.reserve(num.size());
    for (char c : num) {
        while (!st.empty() && k > 0 && st.back() > c) {
            st.pop_back();
            --k;
        }
        st.push_back(c);
    }
    while (k > 0 && !st.empty()) {
        st.pop_back();
        --k;
    }
    // strip leading zeros
    size_t pos = st.find_first_not_of('0');
    string res = (pos == string::npos) ? "" : st.substr(pos);
    return res.empty() ? "0" : res;
}

Java Implementation

class Solution {
    public String removeKdigits(String num, int k) {
        StringBuilder st = new StringBuilder();
        for (int i = 0; i < num.length(); i++) {
            char c = num.charAt(i);
            while (st.length() > 0 && k > 0 && st.charAt(st.length() - 1) > c) {
                st.deleteCharAt(st.length() - 1);
                k--;
            }
            st.append(c);
        }
        while (k > 0 && st.length() > 0) {
            st.deleteCharAt(st.length() - 1);
            k--;
        }
        // strip leading zeros
        int idx = 0;
        while (idx < st.length() && st.charAt(idx) == '0') idx++;
        String res = idx == st.length() ? "" : st.substring(idx);
        return res.isEmpty() ? "0" : res;
    }
}

Complexity

  • Time: Each digit is pushed and popped at most once â‡’ O(N).
  • Space: Stack/string holds up to N digits â‡’ O(N).

Conclusion

  • Use a monotonic increasing stack to favor smaller digits in higher places.
  • Spend remaining removals on the suffix after the main scan.
  • Normalize by stripping leading zeros; return "0" if empty.
  • Runs in linear time with linear auxiliary space.
