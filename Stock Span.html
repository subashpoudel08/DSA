Stock Span Problem | Monotonic Stack

For each trading day, compute how many consecutive previous days had a price less than or equal to today‚Äôs price.

Stack Monotonic Nearest Greater to Left O(N) Arrays
Reading time: ~10 min Keywords: stock span, online stock span, monotonic stack, nearest greater to left, O(N)

Introduction

Given an array of daily stock prices, the span on day i is the number of consecutive days ending at i for which the price was ‚â§ price[i]. Return an array of spans for all days. In the online version, expose next(price) that returns today‚Äôs span as prices stream in.

Summary

  • Maintain a monotonically decreasing stack of price-index pairs.
  • While stack top price ‚â§ current price, pop (those days are covered by current span).
  • If stack becomes empty, span = i + 1; else span = i - top.index.
  • Push the current (price, i) and record span; each element is pushed and popped at most once ‚áí O(N).
  • Space is O(N) in the worst case (strictly decreasing prices).

Example & Breakdown

Example prices: [100, 80, 60, 70, 60, 75, 85]

  • Interpretation: For each day, count how many consecutive days back had price ‚â§ today‚Äôs.
  • Expected spans: [1, 1, 1, 2, 1, 4, 6].
  • Intuition: A higher price ‚Äúabsorbs‚Äù previous lower-or-equal prices until a greater price stops the span.
The span is equivalent to the distance to the nearest greater element to the left (or to start if none).

Brute-Force Baseline

  • For each day i, scan leftward while price[j] ‚â§ price[i] to count consecutive days.
  • Nested loops in the worst case: each day may scan back through all previous days.
  • Time: O(N^2). Space: O(1) extra.

Greedy Criterion & Invariant

Maintain a stack that is strictly decreasing in price from bottom to top. This ensures the top is the nearest day to the left with a price greater than today‚Äôs.

  • Pop while top.price ‚â§ current.price to discard days dominated by today‚Äôs higher price.
  • If the stack is empty, no greater price exists to the left ‚áí span covers all previous days plus today.
  • Otherwise, the nearest greater-to-left day at top.index bounds the span.

Single-Pass Stack Algorithm

  • Initialize empty stack of pairs (price, index) and an empty result list.
  • For each day i from left to right:
    • While stack not empty and stack.top().price ‚â§ price[i], pop.
    • If stack empty ‚áí span = i + 1; else span = i - stack.top().index.
    • Push (price[i], i) and append span to result.
  • Return the result list of spans.

The online variant exposes next(val) that applies the same logic incrementally and returns today‚Äôs span.

  • Day 0 (100): stack empty ‚áí span = 1; push (100,0). result = [1]; stack = [(100,0)]

Pseudocode & Implementations

function stockSpan(prices):
    st = empty stack of pairs (price, index)
    spans = empty list
    for i from 0 to prices.length-1:
        while st not empty and st.top().price <= prices[i]:
            st.pop()
        if st empty:
            span = i + 1
        else:
            span = i - st.top().index
        st.push((prices[i], i))
        spans.append(span)
    return spans

C++ Implementation (Array version)

#include <bits/stdc++.h>
using namespace std;

vector<int> stockSpan(const vector<int>& prices) {
    vector<int> span;
    span.reserve(prices.size());
    // pair: (price, index)
    vector<pair<int,int>> st;
    st.reserve(prices.size());
    for (int i = 0; i < (int)prices.size(); ++i) {
        while (!st.empty() && st.back().first <= prices[i]) st.pop_back();
        int s = st.empty() ? i + 1 : i - st.back().second;
        span.push_back(s);
        st.emplace_back(prices[i], i);
    }
    return span;
}

Java Implementation (Online class)

import java.util.*;

class StockSpanner {
    // pair: int[]{price, index}
    private Deque<int[]> st = new ArrayDeque<>();
    private int index = -1;

    public int next(int price) {
        index++;
        while (!st.isEmpty() && st.peekLast()[0] <= price) {
            st.pollLast();
        }
        int span = st.isEmpty() ? index + 1 : index - st.peekLast()[1];
        st.addLast(new int[]{price, index});
        return span;
    }
}

Complexity

  • Time: O(N). Each day‚Äôs price is pushed and popped at most once.
  • Space: O(N) in the worst case (strictly decreasing prices).

Conclusion

  • Span equals distance to the nearest greater price on the left (or start if none).
  • A monotonically decreasing stack yields a clean O(N) solution.
  • The approach works in batch (array) and online (streaming via next).
  • Worst-case memory occurs on strictly decreasing sequences.
