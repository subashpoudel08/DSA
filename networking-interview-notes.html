Basics & Definitions (Q1–Q6)

Q1. Differentiate between a Router, a Hub, and a Switch
  • Hub: L1 repeater; broadcasts frames to all ports; no filtering; legacy.
  • Switch: L2; forwards by MAC table; isolates collisions; VLAN support.
  • Router: L3; routes by IP; performs NAT, ACLs; connects LAN/MAN/WAN.
Tip: In modern networks, hubs are obsolete; switches dominate at access/aggregation.
Q2. What is a Link?
A physical or wireless connection between nodes (copper, fiber, Wi‑Fi). Links have capacity (bandwidth), latency, and error characteristics.
Q3. What is a Node?
Any device that can send/receive data: host, printer, router, switch.
Q4. What is a Backbone Network?
High-capacity core interconnecting distribution/access networks; often redundant and using higher-speed links (10/40/100G).
Q5. What is Network Topology?
  • Bus: shared medium; a single break can affect many.
  • Star: central hub/switch; easy to manage; single hub failure impact.
  • Ring: each node connects to two neighbors; token-based variants.
  • Mesh: multiple interconnections; resilient; complex.
  • Tree: hierarchical star/bus mix.
  • Daisy chain: linear series; limited resilience.
  • Hybrid: mix of types; common in enterprises.
  • Point-to-Point: dedicated link between two devices.
Q6. What is a LAN?
Local Area Network within limited area; wired (Ethernet) or wireless (Wi‑Fi). Varies by topology, protocols, and media.

Routers & P2P (Q7–Q8)

Q7. What are Routers?
Layer 3 devices at gateways; forward packets by routing table; can be hardware or virtual; often perform NAT, QoS, ACLs.
Q8. What is a Point-to-Point Network?
A direct link between two nodes (e.g., dedicated fiber, serial line); predictable performance; simpler addressing.

OSI Model (Q9–Q10)

Q9. What is the OSI Model?
Conceptual 7-layer model: Physical, Data Link, Network, Transport, Session, Presentation, Application.
Transport is the core for end-to-end reliability and multiplexing.
Q10. Brief description of each OSI layer
  • Physical: bits; cables, optics, RF.
  • Data Link: frames; MAC, switches, ARP.
  • Network: packets; IP routing, ICMP.
  • Transport: segments/datagrams; TCP/UDP, ports.
  • Session: dialogs, checkpoints.
  • Presentation: formats, encryption, compression.
  • Application: HTTP, DNS, SMTP, FTP, etc.
PDU names: Frame (L2), Packet (L3), Segment (TCP), Datagram (UDP).

Access & Services (Q11–Q13)

Q11. What is Anonymous FTP?
Public file access by logging in as anonymous; minimal auth. Modern note: replaced by HTTPS/CDN/object storage for security.
Q12. What is a Network?
Connected devices via physical (twisted pair, fiber, coax) or wireless (Wi‑Fi, microwave). Purpose: share resources, communicate, exchange data.
Q13. What is a Subnet Mask?
32-bit mask splits IP into network and host portions. Network bits = 1, host bits = 0. 
Example: 192.168.1.42/24 → mask 255.255.255.0
Network = 192.168.1.0, Broadcast = 192.168.1.255
Usable hosts = 254 (1–254)
Use CIDR notation /n; avoid assigning network/broadcast addresses.

TCP/IP & Comparison (Q14–Q16)

Q14. TCP/IP Model
4 layers: Link (L2), Internet (IP/ICMP/ARP), Transport (TCP/UDP), Application (HTTP/DNS/SMTP...).
TCP/IP predates OSI; widely used in real systems.
Q15. Difference between OSI and TCP/IP
OSI is 7-layer conceptual reference; TCP/IP is 4-layer practical suite. Reliability is a property of protocols (e.g., TCP) not the model.
Q16. What is a UTP cable?
Unshielded Twisted Pair copper; twists reduce EMI. Categories (Cat5e, Cat6, Cat6a) define bandwidth and max frequency.

Cabling & HTTP/NAT/TCP/UDP (Q17–Q22)

Q17. Max length for a UTP cable
100 meters per Ethernet segment (90 m permanent link + 10 m patch cords typical).
Q18. What is HTTP and its port?
Hypertext Transfer Protocol, stateless, media-independent. Default port 80 (HTTPS 443). Methods, headers, caching define behavior.
Q19. What is NAT?
Translates private↔public addresses. Types: SNAT (source), DNAT (destination), PAT (port address translation). Hides internal IPs, conserves IPv4.
Q20. What is TCP?
Reliable, ordered, connection-oriented transport with flow + congestion control. Segments data, uses ports, 3-way handshake.
Q21. Brief explanation of UDP
Connectionless, best-effort transport; no ordering or retransmissions; ideal for realtime, DNS, QUIC base.
Q22. Differentiate between TCP and UDP
  • TCP: reliable, ordered, ACKs, congestion/flow control.
  • UDP: unordered, no reliability; app-level handling if needed.
  • Use TCP for files, DB, critical RPCs; UDP/QUIC for realtime and low latency.

Routing & Security (Q23–Q26)

Q23. What is RIP?
Routing Information Protocol; distance-vector; metric = hop count; v1 (classful), v2 (CIDR, auth), UDP/520; AD=120.
Q24. What is a Firewall?
Filters traffic per rules; network (perimeter) or host-based; L3–L7; can include IDS/IPS and WAF capabilities.
Q25. What is NOS?
Network Operating System: OS with networking features for services, auth, file/print, clustering (e.g., Linux, Windows Server).
Q26. What is Denial of Service (DoS)?
Overwhelming a target to degrade/deny service. Mitigations: rate limiting, scrubbing, CAPTCHAs, autoscaling, anycast.

Standards & Addressing (Q27–Q31)

Q27. Full form of ASCII
American Standard Code for Information Interchange, 7-bit historical encoding; UTF‑8 supersedes for i18n.
Q28. What is IEEE?
Professional body; 802 standards (Ethernet 802.3, Wi‑Fi 802.11) among many contributions.
Q29. What is a MAC address and why required?
48-bit (or 64-bit EUI‑64) L2 identifier for NICs; used for switching and ARP; unique per interface (with exceptions).
Q30. What is Piggybacking?
In full-duplex protocols, ACK combined with outbound data to reduce frames; used with sliding windows.
Q31. What is DNS?
Translates names ↔ IPs; hierarchical; caching with TTL.

dig A example.com +short

Domains & OSPF/RTT/DHCP (Q32–Q35)

Q32. Domain vs Workgroup
  • Domain: centralized directory (AD); policy and auth managed.
  • Workgroup: peer-to-peer; local users; small networks.
Q33. What is OSPF?
Link-state IGP; Dijkstra SPF; areas (backbone Area 0); fast convergence; metrics via cost; supports VLSM.
Q34. What is Round Trip Time?
Time from send to response; affected by propagation, queueing, processing; measured by ping/traceroute.
Q35. What is DHCP?
Dynamic Host Configuration Protocol; assigns IP, mask, gateway, DNS via DORA (Discover, Offer, Request, Ack); UDP 67/68.

ICMP/Ping/Fiber/Client-Server (Q36–Q39)

Q36. What is ICMP?
Control protocol for errors/diagnostics (e.g., Destination Unreachable, Time Exceeded); not for data transport.
Q37. What is Ping?
Sends ICMP Echo Request; expects Echo Reply; checks reachability/latency; may be blocked by firewalls.
Q38. Advantages of Optical Fiber
Higher bandwidth, lower attenuation, EMI immunity, lightweight, difficult to tap. Single-mode for long distance; multi-mode for short.
Q39. What is a Client/Server network?
Servers provide resources/services; clients request; examples: web, email, print; enables central management.

AV/Ethernet/SLIP/CSMA (Q40–Q43)

Q40. Best place to install antivirus
Install on all endpoints and servers; layered security; central management recommended.
Q41. What do you mean by Ethernet?
L2 LAN technology (IEEE 802.3); frames, MAC, CSMA/CD (legacy); full-duplex switching today.
Q42. What is SLIP?
Serial Line IP; historic encapsulation over serial; superseded by PPP (authentication, error detection, IPCP).
Q43. CSMA/CD vs CSMA/CA
  • CSMA/CD: detect collisions after; used in hub-era Ethernet; obsolete with switches.
  • CSMA/CA: avoid collisions via RTS/CTS/backoff; used in Wi‑Fi.

IPsec/IPv6/Crypto (Q44–Q46)

Q44. Tunnel Mode
IPsec mode encrypting entire IP packet; used for site-to-site VPNs; adds new outer IP header.
Q45. What is IPv6?
128-bit addresses; no NAT needed; SLAAC, Neighbor Discovery; IPSec mandatory support; larger header but more efficient routing.
Q46. Explain RSA briefly
Public-key crypto; slow—used to exchange symmetric keys or sign. Modern TLS 1.3 favors ECDHE key exchange and ECDSA for signatures.

Encoders/Decoders/Sneakernet/Protocol (Q47–Q50)

Q47. What is an Encoder?
Converts data format; often analog→digital (ADC) or compression/codecs (e.g., H.264 encoder).
Q48. What is a Decoder?
Reverse of encoder; digital→analog (DAC) or decode compressed streams.
Q49. What is Sneakernet?
Physically moving storage media between machines; high initial latency but high bandwidth (e.g., shipping drives to cloud provider).
Q50. Components of a Protocol
  • Syntax: data format/structure.
  • Semantics: meaning of fields and codes.
  • Timing: when/rate of transmission; timeouts.
