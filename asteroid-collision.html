Asteroid Collision (LeetCode 735)

Simulate collisions using a stack to determine the final state of asteroids.

Stack Simulation Arrays O(N)

Introduction

Topic: Asteroid Collision

Goal Return the final state after all collisions

Example input: asteroids = [5, 10, -5]

Problem Description

  • Each integer represents an asteroid's size (absolute value) and direction (sign).
  • Positive → moving right; Negative → moving left.
  • Only asteroids moving toward each other can collide (right-moving meets left-moving).
  • When two collide: smaller explodes; if equal, both explode.
  • Asteroids moving in the same direction never meet.
Return the remaining asteroids (in order) after all collisions are resolved.

Collision Rules

Collision can only happen when a right-moving asteroid is before a left-moving asteroid:

top > 0 and a < 0 ⇒ they are heading towards each other.
ConditionOutcome
|top| > |a|Current a explodes, top survives
|top| < |a|top explodes; continue comparing a with new top
|top| == |a|Both explode (pop top; do not push a)

If stack is empty or top is negative, a negative asteroid just moves left without meeting anyone.

Collision Scenarios

  • Same direction: [5, 7, 10], [-5, -7, -10] → no collisions.
  • Opposite directions but no meeting: distances or ordering prevent a collision.
  • Opposite directions and meet: compare sizes to decide explosions.

Example Walkthroughs

[5, 10, -5]
  • Stack: [] → push 5 → [5] → push 10 → [5, 10]
  • See -5: top 10 vs -5 → |10| > |−5| ⇒ -5 explodes
  • Result: [5, 10]
[8, -8]
  • Stack: [] → push 8 → [8]
  • See -8: top 8 vs -8 → |8| == |−8| ⇒ both explode
  • Result: []
[10, 2, -5]
  • Stack: [] → [10] → [10, 2]
  • See -5: compare 2 vs -5 → |2| < |−5| ⇒ pop 2 → [10]
  • Now 10 vs -5 → |10| > |−5| ⇒ -5 explodes
  • Result: [10]
[-2, -1, 1, 2]
  • Negatives move left; positives move right; ordering prevents meetings
  • Result: [-2, -1, 1, 2]

Algorithm (Stack)

  1. Initialize empty stack.
  2. For each asteroid a:
    • If a > 0 or stack empty or top < 0 → push a.
    • Else (a < 0 and top > 0):
      • While stack not empty and top > 0 and top < |a|: pop top (top explodes).
      • After loop:
        • If stack empty or top < 0: push a (no more collisions possible).
        • Else if top == |a|: pop top (both explode); do not push a.
        • Else (top > |a|): current a explodes; do nothing.
  3. Return the stack.

Each asteroid is pushed and popped at most once → linear time.

Pseudocode (Python-like)

def asteroidCollision(asteroids):
    stack = []
    for a in asteroids:
        if a > 0 or not stack or stack[-1] < 0:
            stack.append(a)
        else:  # a < 0 and stack[-1] > 0
            while stack and stack[-1] > 0 and stack[-1] < abs(a):
                stack.pop()
            if not stack or stack[-1] < 0:
                stack.append(a)
            elif stack[-1] == abs(a):
                stack.pop()  # both explode
            # else: |top| > |a|, a explodes (do nothing)
    return stack

Time & Space Complexity

  • Time: O(N) — each asteroid is pushed/popped at most once.
  • Space: O(N) — worst-case all asteroids remain on stack.

Conclusion

  • Core idea: Use a stack to simulate head-on collisions.
  • Rule: Collide only when right-moving meets left-moving.
  • Outcome: Clean, linear-time simulation with clear cases.
